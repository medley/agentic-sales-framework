
# Agentic Sales Framework - System Architecture

## Overview

The Agentic Sales Framework is a four-layer Claude Code-native personal sales operating system that separates company data, execution logic, portable knowledge, and tooling documentation. Raw inputs (playbooks, transcripts, battlecards) flow through a conversion pipeline into structured knowledge that feeds an AI sales coach. The coach skill reads deal context from Obsidian notes, applies methodology-aware logic, and generates stage-specific artifacts (emails, agendas, briefings). Claude Code orchestrates the reasoning and file operations using native `.claude/` skills, while Obsidian serves as the workspace UI. The execution layer (skills, hooks) requires Claude Code, while the knowledge layer (methodologies, templates, docs) remains portable across companies.

1. SEPARATION OF CONCERNS
   Company data (sample-data/), execution logic (.claude/), and portable knowledge
   (Framework/) are strictly separated. Each layer has clear ownership and purpose.

2. CLAUDE CODE-NATIVE
   Built for Claude Code's skill system. Skills leverage native .claude/ structure
   for auto-discovery, hooks, and settings. Requires Claude Code.

3. KNOWLEDGE PORTABILITY
   Framework/ layer (methodologies, templates, docs) remains tool-agnostic. Can be
   used with any LLM tool or shared across companies.

4. METHODOLOGY AGNOSTIC
   Works with MEDDPICC, Sandler, Challenger, SPIN, or custom methodologies.
   Coach skill reads adapter files to apply appropriate logic dynamically.

5. CONTEXT PRESERVATION
   Everything links back to source. Coach skill references specific knowledge
   files. Deal notes link to personas, battlecards, artifacts. Full provenance.

6. ZERO MANUAL COPYING
   Claude Code writes directly to Obsidian workspace. Generated content appears
   in-place. No copy/paste to external systems (unless user chooses to).

7. SAFETY THROUGH HOOKS
   .claude/hooks/ prevent writing to wrong locations. Framework/ is read-only
   (except System/ docs). sample-data/input/ is immutable. Enforced automatically.

8. INCREMENTAL ADOPTION
   Start with one skill (convert or coach), one deal. Add complexity as value
   proves out. Framework grows with you.
